# 🧾 Invoice Ninja Octane

**URL:** [invoice.siyalude.io](https://invoice.siyalude.io)
**Install Path:** `/opt/invoiceninja`

Invoice Ninja is a powerful, open-source invoicing, tracking, and billing application for freelancers and businesses. This deployment uses the **Octane** version for high-performance PHP execution via Swoole/RoadRunner.

## 🚀 Deployment

The deployment is fully automated via `setup-invoiceninja.sh`.

```bash
# Deploy to iperamuna
rsync -avz ./InvoiceNinja-Octane/ iperamuna:/opt/invoiceninja/
ssh iperamuna "sudo bash /opt/invoiceninja/setup-invoiceninja.sh"
```

## 🛠️ Stack Components

| Service | Image | Description |
|---|---|---|
| **App** | `invoiceninja/invoiceninja-octane` | Core application running on Octane |
| **DB** | `mariadb:10.11` | Primary database |
| **Redis** | `redis:alpine` | Cache and session management for Octane |
| **PDF** | `invoiceninja/snapp-pdf` | Local PDF generation engine |

## ⚙️ Configuration

The configuration is managed via `.env`. Key variables:

- `APP_URL`: The public URL (https://invoice.siyalude.io)
- `APP_KEY`: Laravel application key (generated by setup script)
- `SNAPP_PDF_ENABLED`: Enabled for local PDF generation
- `TRUSTED_PROXIES`: Set to `*` for Nginx reverse proxy compatibility

## 📋 Common Operations

```bash
# View live logs
ssh iperamuna "cd /opt/invoiceninja && docker compose logs -f"

# Restart application
ssh iperamuna "cd /opt/invoiceninja && docker compose restart"

# Run artisan commands
ssh iperamuna "cd /opt/invoiceninja && docker compose exec server php artisan <command>"
```

## ⚠️ Troubleshooting

### PDF Generation Issues
If PDFs fail to generate, ensure the `snapp-pdf` container is running and reachable by the `server` container on the internal network.

### Database Connection
On first run, the database might take a moment to initialize. The setup script includes a delay, but if migrations fail, you can run them manually:
`docker compose exec server php artisan migrate --force`
